Pin Module

Pin Selection Window

Summary: The Pin Selection Window allows the user to select a certain pin for a specific peripheral.
Description: This window has a table with all the pins and all the current loaded peripherals. A pin can be assigned to a specific peripheral by "locking" it. When a pin is locked to a function, it is connected to the selected peripheral and has the selected function. If a pin is locked to a new peripheral but aready has a selected function, the pin is disconnected from the old peripheral and reassigned to the new peripheral.
Additional Information: A pin can be either an input or an output. Some peripherals only have input or output lines, depeding on the function to be linked to a pin.
Availability: The Pin Selection Window is always available.

Additional pin functions window

Summary: This window allows you to set additional pin functions
Description: Once a pin is assigned to a peripheral, a new row in the functions window table shows up. Here you can set additional pin settings such as Custom pin name, analog or digital function, input or output, weak pull-up, open drain, and interrupt on change settings.
Additional Information: The pin functions such as analog/digital and input/output actually control the value of the ANSEL and TRIS register. The same for the weak pull-up functions (WPU register) and open drain.
Availability: The Pin Functions Window is always available.

Package View

Summary: The package view shows current processor package selected along with the pin functions.
Description: The package view provides a quick overview of the microcontroller capsule that is currently used in the project.
Additional Information: The package view provides a quick overview of the microcontroller capsule that is currently used in the project. The graphical representation of the package is shown as viewed from top. All the pins are visible along with their names and their currrent selected functions.
Availability: The Package View Window is always available.
