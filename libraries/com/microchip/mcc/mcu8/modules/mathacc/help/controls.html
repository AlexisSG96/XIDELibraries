Math Accelerator with Proportional-Integral-derivative (PID) Module

Mode

Summary: Selects between Math and PID functions.
Description: The MATHACC module can operate in two modes: Add/Multiply or PID. Selecting one mode disables the control section for the other.
Availability: Always available.

Operation

Summary: Selects an Add/Multiply combination.
Description: The Add/Multiply mode can be used in several ways, depending on the mathematical function that needs to be performed. This mode makes use of 3 parameters (A, B, C) you can define for your application. Referring to the Result = (A + B) * C equation, the Operation drop-down allows you to:

1. Add and Multiply - use the equation as above

2. Add only, where C = 1

3. Multiply only, where A = 0

The Result of the Operation is displayed underneath the A, B and C parameters.
Additional Information: All Operation changes trigger a recalculation of the Result value.
Availability: Conditioned by the Mode selection (disabled in PID mode).

Inputs

Summary: Defines the A, B and C parameters as signed or unsigned integers.
Description: Changing this drop-down modifies the input range of the A, B and C parameters. If the Inputs are unsigned, the range is [0..65535], if they are signed (2's complement), the range is [-32768, 32767].
Availability: Conditioned by the Mode selection (disabled in PID mode).

Parameter A

Summary: Parameter A in the Result = (A + B) * C equation
Description: This is an integer parameter you need to define when using Add and Multiply or Only Add operations. Its value is 0 and cannot be modified in Only Multiply mode. Each update on this parameter will recalculate the Result value displayed.
Additional Information: Selecting an input out of the specified range will trigger a warning in the Notifications tab. If the input contains non-integers or non-numeric characters the input box becomes invalid and it will blink in red until the input becomes valid again.
Availability: Conditioned by the Mode selection (disabled in PID mode) and the Operation (disabled in Only Multiply operation).

Parameter B

Summary: Parameter B in the Result = (A + B) * C equation
Description: This is an integer parameter you need to define for all Add/Multiply operations. Each update on this parameter will recalculate the Result value displayed.
Additional Information: Selecting an input out of the specified range will trigger a warning in the Notifications tab. If the input contains non-integers or non-numeric characters the input box becomes invalid and it will blink in red until the input becomes valid again.
Availability: Conditioned by the Mode selection (disabled in PID mode).

Parameter C

Summary: Parameter C in the Result = (A + B) * C equation
Description: This is an integer parameter you need to define when using Add and Multiply or Only Multiply operations. Its value is 1 and cannot be modified in Only Add mode. Each update on this parameter will recalculate the Result value displayed.
Additional Information: Selecting an input out of the specified range will trigger a warning in the Notifications tab. If the input contains non-integers or non-numeric characters the input box becomes invalid and it will blink in red until the input becomes valid again.
Availability: Conditioned by the Mode selection (disabled in PID mode) and the Operation (disabled in Only Add operation).

Parameter Kp

Summary: Defines the Proportional component of the PID Controller
Description: The PID Controller equation can be expressed as: u(t) = Kp * e(t) + Ki * ∫e(t)dt + Kd * (de(t)/dt), where e(t) = error (difference between the Set Point and the actual Output). Kp determines the Proportional/current error and can be defined in the user interface as a float number between 0 and 65535. This parameter, together with Ki and Kd, is converted into other 3 16-bit parameters (K1, K2 and K3) that can be written to directly as registers. The MATHACC graphical interface abstractizes the K1, K2, K3 calculation, so that you are required to provide only Kp, Ki and Kd parameters.
Additional Information: Selecting an input out of the specified range or containing non-numeric characters will be considered invalid and the input box will blink in red until the input becomes valid again.
Availability: Conditioned by the Mode selection (disabled in Add/Multiply mode).

Parameter Ki

Summary: Defines the Integral component of the PID Controller
Description: The PID Controller equation can be expressed as: u(t) = Kp * e(t) + Ki * ∫e(t)dt + Kd * (de(t)/dt), where e(t) = error (difference betwen the Set Point and the actual Output). Ki determines the sum of all previous errors and can be defined in the user interface as a float number between 0 and 65535. This parameter, together with Kp and Kd, is converted into other 3 16-bit parameters (K1, K2 and K3) that can be written to directly as registers. The MATHACC graphical interface abstractizes the K1, K2, K3 calculation, so that you are required to provide only Kp, Ki and Kd parameters.
Additional Information: Selecting an input out of the specified range or containing non-numeric characters will be considered invalid and the input box will blink in red until the input becomes valid again.
Availability: Conditioned by the Mode selection (disabled in Add/Multiply mode).

Parameter Kd

Summary: Defines the Derivative component of the PID Controller
Description: The PID Controller equation can be expressed as: u(t) = Kp * e(t) + Ki * ∫e(t)dt + Kd * (de(t)/dt), where e(t) = error (difference betwen the Set Point and the actual Output). Kd determines the last error (current vs. previous output) and can be defined in the user interface as a float number between 0 and 65535. This parameter, together with Kp and Ki, is converted into other 3 16-bit parameters (K1, K2 and K3) that can be written to directly as registers. The MATHACC graphical interface abstractizes the K1, K2, K3 calculation, so that you are required to provide only Kp, Ki and Kd parameters.
Additional Information: Selecting an input out of the specified range or containing non-numeric characters will be considered invalid and the input box will blink in red until the input becomes valid again.
Availability: Conditioned by the Mode selection (disabled in Add/Multiply mode).
