Direct Memory Access (DMA)

DMA Module

Summary: The Direct Memory Access (DMA) module is designed to service data transfers between different memory regions directly without intervention from the CPU. By eliminating the need for CPU-intensive management of handling interrupts intended for data transfers
Description: The DMA Module can be easily enabled by configuring the following steps in order:
  • Source Address Memory
  • Destination Address Memory
  • Address Mode
  • Message Size/Counters
  • Start Interrupt Source Selection
  • Abort Interrupt Source Selection
  • Enable DMA Module
Availability: Always available

Source Address Memory

Summary: Specifies memory region DMA can access for copying of data
Description: The DMA Module has read access to the below memory locations.
  • Program Flash Memory (PFM)
  • Data EEPROM (Data EEPROM)
  • General Purpose Registers (GPR)
  • Special Function Registers (SFR)
Availability: Always available

Destination Address Memory

Summary: Specifies memory region DMA can access for writing of data
Description: The DMA Module has write access to the below memory locations.
  • General Purpose Registers (GPR)
  • Special Function Registers (SFR)
Availability: Always available

Address Mode

Summary: Allows various data transfer modes by controlling increment or decrement of address pointer registers.
Description: Source and Destination address pointer registers are modified after each transaction based on the Address Mode selection bits. Following are different type of transfer modes possible
  • N: 1 This type of transfer is common when sending predefined data packets (such as strings) through a single interface point (such as communications modules transmit registers).
  • N: N This type of transfer is useful for moving information out of the Program Flash or Data EEPROM to SRAM for manipulation by the CPU or other peripherals.
  • 1: 1 This type of transfer is common when bridging two different modules data streams together (communications bridge).
  • 1: N This type of transfer is useful for moving information from a single data source into a memory buffer
Availability: Always available

Message Size/Counters

Summary: A complete DMA transaction process consists of one or more messages. The size registers determine how many transactions are in a message.
Description: When a DMA transfer is initiated the size registers are copied to corresponding counter registers that control the duration of the message. Both the source and destination counters are simultaneously decremented by one after each transaction.
Availability: Always available

Start Interrupt Source Selection

Summary: The DMA can initiate data transactions by selecting Hardware Trigger
Description: A Hardware trigger is an interrupt request from another module sent to the DMA with the purpose of triggering DMA transaction.
Availability: Always available

Abort Interrupt Source Selection

Summary: The DMA can abort data transactions by selecting Hardware Trigger
Description: A Hardware trigger is an interrupt request from another module sent to the DMA with the purpose of aborting DMA transaction.
Availability: Always available

Enable DMA Module

Summary: Enable DMA Module initiates DMA Message Transfer
Description: After enabling DMA Module, it waits for the Hardware or Software trigger to start DMA message transfer
Availability: Always available
