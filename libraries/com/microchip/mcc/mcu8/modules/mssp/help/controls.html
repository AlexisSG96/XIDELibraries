Master Synchronous Serial Port

Mode

Summary: Selects the serial protocol and its function within the application.
Description: The MSSP module multiplexes the same hardware interface for two serial communication protocols : the Serial Parallel Interface (SPI) and the Inter-Integrated Circuit (I2C). Using the MSSP Mode drop-down you can select between the two protocols and also the operation method within the application (Master or Slave). Depending on the Mode selection, the MSSP graphical interface is populated with the appropriate settings and the Pin Manager window will show only the corresponding MSSP pin functions.
Additional Information: The mode selection will determine the name of the generated MSSP source code files.
Availability: Always available.

I2C Master - Baud Rate Generator Value

Summary: Defines the I2C Clock Frequency
Description: This is an 8-bit value you can use to set the frequency of the I2C communication. The frequency is calculated using the I2C_period = (I2C_Baudrate_Value + 1) / Fosc equation, displayed the I2C Clock section and refreshed each time there's an update on the Baud Rate Generator value.
Additional Information: You can use either a decimal or a hexadecimal input format. Using a value that's out of the specified range will trigger a warning message in the Notifications tab. Using non-integer or non-numeric characters will invalidate the input box content which will blink red until the input is valid again.
Availability: Conditioned by the MSSP mode selection (only available in I2C Master mode).

I2C Slave - Serial Port Mode

Summary: Selects between several I2C Slave communication modes.
Description: The I2C Slave mode of the MSSP module can be configured to communicate in several ways, depending on the slave address bitsize and the usage of interrupts. Although supported by the hardware, some of the options may not be available in the MCC interface of the MSSP module. These options are not currently supported in MCC.
Additional Information: The ranges of the Slave Address and Slave Mask inputs are adjusted automatically based on the Serial Port Mode selection.
Availability: Conditioned by the MSSP mode selection (only available in I2C Slave mode).

I2C Slave - Address

Summary: The address of the microcontroller used as slave device
Description: In the I2C communication, the Master node is required to know the address of the Slave node in order to communicate with it. Depending on the addressing mode selected in the Serial Port Mode drop-down, the slave address range can be changed.
Additional Information: You can use either a decimal or a hexadecimal input format. Using a value that's out of the specified range will trigger a warning message in the Notifications tab. Using non-integer or non-numeric characters will invalidate the input box content which will blink red until the input is valid again.
Availability: Conditioned by the MSSP mode selection (only available in I2C Slave mode).

I2C Slave - Mask

Summary: Address Mask for the Shift Register.
Description: The Transmit/Receive Shift Register is by definition not readable or writable. Therefore, in order to have access to it during an address comparison, a secondary mask register is used. On a bit level, during the address comparison process, each bit from the address received in the Shift Register (accessed through the Address Mask) is compared to the Address Register.
Additional Information: You can use either a decimal or a hexadecimal input format. Using a value that's out of the specified range will trigger a warning message in the Notifications tab. Using non-integer or non-numeric characters will invalidate the input box content which will blink red until the input is valid again.
Availability: Conditioned by the MSSP mode selection (only available in I2C Slave mode).

SPI Mode

Summary: SPI signal characteristics
Description: Based on several signal characteristics, such as signal Clock Polarity or the Clock Edge, a certain SPI Mode is selected in the MSSP configuration registers. The actual SPI Mode value is also displayed in the Easy Setup tab.
Availability: Conditioned by the MSSP mode selection (only available in SPI Master or SPI Slave modes).

SPI Master - Clock Source

Summary: Clock Signal for the SPI.
Description: You can select between several clock sources, including some of their variations. One particular option is the divider option which makes use of the SSPADD register. Enabling this option will also enable the Clock Divider input box, so you can specify the Divider value. For each update on the Clock Source, the SPI Clock label is updated with the calculated SPI Frequency.
Additional Information: If you select a clock source which is based on a certain peripheral signal (i.e. TMR2), a warning will be thrown in the Notifications tab letting you know that you need to enable and configure that peripheral.
Availability: Conditioned by the MSSP mode selection (only available in SPI Master mode).

SPI Master - Clock Divider

Summary: Division factor for one of the SPI Clock Sources.
Description: This is an 8-bit value you can use with the SSPADD Clock Source option of the SPI Master. Typically, the formula is: clock = Fosc / (4 * (SSPADD+1)). For each update on the Clock Divider, the SPI Clock label is updated with the calculated SPI Frequency.
Additional Information: You can use either a decimal or a hexadecimal input format. Using a value that's out of the specified range will trigger a warning message in the Notifications tab. Using non-integer or non-numeric characters will invalidate the input box content which will blink red until the input is valid again.
Availability: Conditioned by the MSSP mode selection (only available in SPI Master mode).
