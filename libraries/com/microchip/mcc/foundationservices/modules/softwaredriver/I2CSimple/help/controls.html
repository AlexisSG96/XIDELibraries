Driver description

The I2C Simple driver operates in Master mode. This is a blocking asynchronous code. The APIs in this driver can be used for simple I2C read and write operations.

The I2C Simple driver is configured using MCC. If several I2C hardware instances are available on the device, the user can select which I2C instance the driver shall use.

The I2C Simple is a Master only configuration. When I2C Simple is loaded, it also loads the corresponding instance of the I2C peripheral under Project resources>Peripherals Tab. The user can set options dependent on the hardware used to implement the I2C Simple driver here. For example, an option may allow changing the baud rate used to drive the underlying I2C hardware.



The **I2C Simple** driver provides the following functionality:
  • Read 1 byte of data from a register location
  • Read 2 byte of data from a register location
  • Read N bytes of data from a register location
  • Read N bytes of data
  • Write 1 byte of data to a register location
  • Write 2 byte of data to a register location
  • Write N bytes of data

Hardware dependencies

The I2C Simple driver usually uses some sort of hardware that implements I2C functionality.Different MCUs have I2C hardware with different names and functionalities, such as MSSP, I2C, TWI etc. In MCC, the user selects a device and adds the I2C Simple driver. A device may have several possible hardware resources available for the driver, such as MSSP1, MSSP2 etc. In this case the user must select which one to use.
