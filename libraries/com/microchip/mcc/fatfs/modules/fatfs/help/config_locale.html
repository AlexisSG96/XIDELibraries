FatFS Locale, Namespace, and Buffering Configuration

Configures the size of the sector sizes for the library, thus impacting the RAM footprint. The section also contains information about the formatting of the strings used to reference the drives and their files and folders.

Language Code Page

Description: This sets the platforms language code page. All selected code pages support basic ASCII.
This corresponds to the FF_CODE_PAGE option.

Volume ID Format

Description: Select the how the volumes ids are referenced in code. This is used when a function wants to reference a file or folder via absolute path.
This corresponds to the FF_STR_VOLUME_ID configuration option.

Minimum Sector Size

Description: Defines the minimum sector size. The vast majority of FAT devices will use 512. It is recommended to keep the minimum to 512.
This corresponds to the FF_MIN_SS option.

Maximum Sector Size

Description: Defines the maximum sector size supported. The vast majority of FAT devices will use a 512 byte sector size. This option will be the largest single contributor to the RAM footprint of the library as it will determine how large the sector cache objects are which dominate the RAM usage.
This corresponds to the FF_MAX_SS option.

File Data Buffer

Description: This option switches between a each file having their own data buffer ( 0 - Normal ) and all files sharing a single data buffer (1 - Tiny). This has a implication on the RAM size when more than one file object is allocated at a time. Selecting the tiny option will result in smaller RAM size, but would have higher latency when multiple files are used at the same time as the file system would have to read/write back to the physical media when switching between the files.
This corresponds to the FF_FS_TINY option.
