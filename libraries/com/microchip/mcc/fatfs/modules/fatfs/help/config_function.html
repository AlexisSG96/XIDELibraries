FatFS Function Configuration

Allows the user to select which features of the FatFs library to include allowing the user to remove unused features to reduce code footprint. This section also allows user to configure how some of the functions operate.

Read/Write vs Read Only

Summary: Enables/disables the ability to write to a drive.
Description: Allows the user to disable/enable writing the the attached driver. Removing write support by switching to "Read-Only" will reduce the code footprint of the library. It will also remove the code that does the writes from the code base helping prevent accidental writes to the drive. Corresponds to the FF_FS_READONLY configuration option in FatFs.

Function Optimization

Summary: Enables/disables functions in the FatFs library to save code space.
Description: There are 4 options for this parameter. Each level progressively increases the number of functions removed. For example, when selecting option 2, f_stat(), f_getfree(), f_unlink(), f_mkdir(), f_truncate(), f_rename(), f_opendir(), f_readdir(), f_closedir() are all removed.
This corresponds to the FF_FS_MINIMIZE option.

String Options

Summary: Determines if the string functions are included and how they operate.
Description: There are 3 options for this parameter. Option 0 disables the string functions: f_putc(), f_puts(), f_gets(), and f_printf(). Options 1 and 2 enable the string functions and select how new lines are handled.
This corresponds to the FF_USE_STRFUNC option.

Enable Find Function

Summary: Enables/Disable the find functions.
Description: Enables/Disables the find functions in the firmware: f_findfirst() and f_findnext(). This option is only available when the functions optimization option is set to 0 or 1. This feature is disabled on options 2 and 3.
This corresponds to the FF_USE_FIND option.

Enable Make FAT Volume Function

Summary: Enables/Disables the make FAT volume function in the firmware.
Description: Enables/Disables the make FAT volume function in the firmware. This function allows for the caller to format the drive with a FAT partition.
This corresponds to the FF_USE_MKFS option.

Enable Fast Seek

Summary: Enables/Disables the fast seek functions in the firmware.

Description: Enables/Disables the fast seek functions. These functions allow for faster seeking times in a file by reducing the number of times hardware needs to be accessed by caching information in RAM. Enabling this option will decrease seek times but increase code and RAM usage.
This corresponds to the FF_USE_FASTSEEK option.

Enable Fast Seek

Summary: Enables/Disables the fast seek functions in the firmware.
Description: Enables/Disables the fast seek functions. These functions allow for faster seeking times in a file by reducing the number of times hardware needs to be accessed by caching information in RAM. Enabling this option will decrease seek times but increase code and RAM usage.
This corresponds to the FF_USE_FASTSEEK option.

Enable Expand Function

Summary: Enables/Disables the f_expand() function in the firmware.
Description: Enables/Disables the f_expand() function in the firmware. This function allocates drive space to a file without writing data.
This corresponds to the FF_USE_EXPAND option.

Enable Meta Control Function

Summary: Enables/Disables the meta control functions, f_chmod() and f_utime(), in the firmware.
Description: Enables/Disables the meta control functions, f_chmod() and f_utime(), in the firmware. This option is unavailable in read-only mode.
This corresponds to the FF_USE_CHMOD option.

Enable Label Function

Summary: Enables/Disables the label functions, f_getlabel() and f_setlabel(), in the firmware.
Description: Enables/Disables the label functions, f_getlabel() and f_setlabel(), in the firmware.
This corresponds to the FF_USE_LABEL option.

Enable Forward Function

Summary: Enables/Disables the forward function in the firmware.
Description: Enables/Disables the forward function in the firmware. The f_forward() function allows a file to be read directly into a data stream.
This corresponds to the FF_USE_FORWARD option.

Relative Path Support


Summary: Enable/disable support for relative paths.
Description: This corresponds to the FF_FS_RPATH option. Enabling relative path allows files to be specified by relative path from the current working directory rather than absolute path. If relative path is disabled, all files must be accessed by the full absolute path. Disabling these functions removes the functionality and corresponding functions from the code base thus reducing the code by removing functionality.

Get Current Working Directory Option


Summary: Enable/disable support for the get current working directory function (f_getcwd).
Description: This corresponds to the FF_FS_RPATH option. Enabling this feature adds the f_getcwd function to the code base and increases the code size accordingly. This feature requires the relative path support to be enabled.

Multiple Partition Support

Description: Enable/Disable multiple partition support. When disabled only one partition on a drive is used. When enabled, drives with multiple partitions will have each partition bound.
This corresponds to the FF_MULTI_PARTITION configuration option.

ATA Trim Support

Description: Enable/Disable support for the ATA-Trim command.
This corresponds to the FF_USE_TRIM configuration option.

Enable Full Scan for Free Disk Space

Description: On FAT32 based drives there is a sometimes a FSINFO object that is stored to the drive that contains information about the drive, including the available free size of the drive. Users that need to know the available size of the drive have two options on these types of drives.

  1. Trust the information in the FSINFO object. This option is much quicker but the information in the FSINFO object might not always be correct.
  2. Rescan the drive and calculate the actual free space available. This option is much slower but will always produce accurate results.


This corresponds to bit 0 in the FF_FS_NOFSINFO configuration option.

Enable Full Scan for Next Available Cluster

Description: On FAT32 based drives there is a sometimes a FSINFO object that is stored to the drive that contains information about the drive, including the next available data cluster on the drive. Users that need to know the available size of the drive have two options on these types of drives.

  1. Trust the information in the FSINFO object. This option is much quicker but the information in the FSINFO object might not always be correct.
  2. Rescan the drive and calculate the actual next cluster available. This option is much slower but will always produce accurate results.


This corresponds to bit 1 in the FF_FS_NOFSINFO configuration option.

File Lock Protection Enable

Description:
This determines if the file lock projection is enabled. When disabled, it is the responsibility of the user to make sure that a file/folder isn't opened twice in write mode. If a file is opened twice in write mode, corruption of the file/folder can occur. When enabled, the file system will automatically block opening the same file/folder twice in write mode. Once a file is opened in write mode, it can't be opened a second time until the first open file instance is closed. This automatically prevents file corruption, but increases the code footprint and RAM usage. This corresponds to the FF_FS_LOCK option.

File Lock Maximum Files

Description:
When the file lock feature is enabled, this determines how many files/folders can be opened at the same time. For example, if this field is set to 10, then 10 files can be opened at once. After that, the file locking system will not allow new files to be opened until other files are closed. Increasing this number increases the RAM footprint.

This corresponds to the FF_FS_LOCK option.
